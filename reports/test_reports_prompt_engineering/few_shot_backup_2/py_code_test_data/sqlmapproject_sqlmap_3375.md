Vulnerability: SQL Injection
Severity: Critical
CWE: CWE-89
Location: Multiple instances using `inject.checkBooleanExpression()`
Description: The function relies heavily on executing raw, complex SQL queries via `inject.checkBooleanExpression()`. While this functionality is intended for database fingerprinting, the pattern of constructing and executing multiple boolean expressions based on system state variables (e.g., `QUARTER(NULL) IS NULL`, `SESSION_USER() LIKE USER()`) creates a high risk of SQL Injection if any part of the query construction were to incorporate untrusted or user-controlled input. An attacker could potentially manipulate inputs or constants used in these expressions to bypass logic, extract sensitive information (like schema names, table contents), or execute arbitrary commands against the underlying database.
Remediation: Since this function's core purpose is executing complex SQL for detection, standard parameterized queries may not fully address all risks. However, any variable or configuration value that contributes to the construction of these SQL strings must be rigorously validated and sanitized (whitelisting allowed characters/keywords). If possible, the underlying database connector library should provide a secure API layer specifically designed for safe execution of system-level checks, ensuring that no user input can break out of the intended query context.
